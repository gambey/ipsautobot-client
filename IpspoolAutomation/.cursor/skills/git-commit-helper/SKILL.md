# Git 提交说明助手（简体中文）

## 何时使用

- 用户需要**一条或多条**可直接粘贴到 `git commit` 的说明。
- 用户展示或隐含**本次改动的意图**（功能、修复、重构等）。
- 需要基于 **diff / 文件列表** 归纳变更时。

## 输出语言

- **标题与正文默认使用简体中文**；类名、API、路径、英文标识符可保留英文。
- 若用户明确要求英文或其它语言，以用户为准。

## 工作方式

1. **尽量读取事实**：必要时运行或建议用户运行 `git status`、`git diff --staged`（或 `git diff`）以确认实际变更范围；若无法执行命令，则根据对话与已打开文件推断，并注明依据。
2. **一条提交聚焦一事**：若变更混杂多主题，建议拆分为多次提交，并分别给出说明。
3. **说明要写清「做了什么、为什么」**，避免仅写「修改」「更新」等空泛词。

## 推荐格式（可选用）

### 简短一行（适合小改动）

```text
<类型>：<简短概括（中文）>
```

常见类型：`feat`、`fix`、`refactor`、`docs`、`chore`、`perf`、`test`（可与团队约定一致）。

### 带正文（适合较大改动）

```text
<类型>：<概括（中文）>

- 要点一（中文）
- 要点二（中文）
```

## 反例与正例

- ❌ `fix bug` / `update` / `修改代码`
- ✅ `fix(提现)：纠正序号在 JSON 持久化后未刷新的问题`
- ✅ `feat(签到)：定时签到到达后仅当日自动执行一次`

## 与项目约定

- 本仓库 `.cursor/rules` 中若存在「提交说明使用中文」类规则，与其保持一致。
